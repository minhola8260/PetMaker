## 1. 프로젝트 개요

### 1.1 프로젝트 목표
- 단순하고 지루할 수 있는 걷는 행위와 같은 야외활동을 '새로운 나만의 펫과의 만남'이라는 모험성있고 탐험적인 게임적 요소로 치환하여 사용자의 야외활동을 유도한다. 사용자의 실시간 환경(위치, 날씨, 시간대)을 분석하여 AI가 해당 환경에 적합한 디지털 펫(이미지 및 상세 정보)을 생성하고, 이를 수집하는 탐험 기반의 위치 애플리케이션을 제공한다.

### 1.2 대상 사용자
- 걷기 운동과 같은 움직임과 야외 활동에 동기 부여가 필요한 사람
- 정형화된 캐릭터 수집을 넘어 나만의 고유한 캐릭터 수집을 즐기는 사람(게이머)

### 1.3 핵심 가치
- 기존 수집형 게임의 정해진 캐릭터(정형화된, 정적인 데이터)에서 벗어나 사용자의 실시간 환경(위치, 날씨, 시간대, 기온 등) 맥락에 따라 AI를 통해 매번 창의적이고 새로운 나만의 특별한 고유한 캐릭터(펫)를 생성하여 무한한 수집의 재미와 특별한 소유감을 제공한다.

## 2. 기능 요구사항 (Functional Requirements)

> 시스템이 *무엇을 하는가*를 정의한다. 
> 문장 패턴: "사용자는 ~할 수 있다" / "시스템은 ~한다"

### FR-01: [실시간 환경(위치, 날씨, 시간대) 정보 수집 및 표시]

- **설명**: 사용자는 앱의 메인 화면에서 현재 위치의 날씨, 기온, 시간대 정보를 직관적인 UI로 확인할 수 있다.
- **수용 조건**:
  - [x] GPS를 통해 현재 위치 정보를 수집한다
  - [x] OpenWeatherMap API를 통해 날씨 정보(맑음, 비, 눈, 흐림)를 수집한다
  - [x] 수집된 날씨 상태에 맞는 배경 그래픽(그라디언트, 애니메이션)이 메인 화면에 렌더링된다
  - [x] 기온 텍스트 및 시간대 구분(아침, 오후, 저녁, 밤)이 실시간으로 화면에 업데이트되어 표시된다
  - [x] 현재 시간이 초 단위로 업데이트되어 표시된다
  
### FR-02: [환경 기반 펫 생성 가능 여부 판정]

- **설명**: 시스템은 실시간 환경 데이터의 조건이 충족되었을 때 내부 알고리즘에 따른 특정 확률로 펫 생성 가능 여부를 판정하고, 활성화 상태를 사용자에게 제공한다.
- **수용 조건**:
  - [x] 10초마다 펫 등장 확률을 체크한다 (50% 확률)
  - [x] 펫 생성이 가능할 때 "새로운 펫 발견!" 버튼이 반짝이는 애니메이션과 함께 메인 화면에 표시된다
  - [x] 펫 생성이 불가능할 때는 해당 버튼이 화면에서 숨겨진다
  - [x] 펫 생성 버튼에는 Sparkles 아이콘과 펄스 애니메이션이 적용된다

### FR-03: [AI 기반 고유 펫 생성]

- **설명**: 사용자가 활성화된 펫 생성 기능을 실행하면, 시스템은 수집된 환경 데이터를 LLM API에 전달하여 조건에 최적화된 고유한 펫을 생성한다.
- **수용 조건**:
  - [x] Google Gemini API에 현재 날씨, 기온, 시간대, 위치 데이터를 프롬프트 형태로 전달한다
  - [x] 해당 환경 맥락에 어울리는 펫의 이름, 상세 외형 묘사, 특징, 성격 텍스트를 반환받는다
  - [x] 펫 생성 중에는 로딩 화면이 표시되며, Sparkles 아이콘이 회전 애니메이션과 함께 나타난다
  - [x] 생성 완료 시 펫 등장 이펙트(별 파티클 효과)와 함께 펫 정보가 화면에 출력된다
  - [x] 펫 정보는 이름, 외형 묘사, 성격, 특성 뱃지로 구성된다

### FR-04: [펫 수집 및 영구 저장]

- **설명**: 사용자는 생성 완료된 펫을 로컬 DB에 영구 저장하고, 도감 화면에서 자신이 수집한 기록들을 언제든 열람할 수 있다.
- **수용 조건**:
  - [x] "도감에 수집하기" 버튼 클릭 시 펫의 정보가 로컬 DB(Room/SQLite)에 저장된다
  - [x] 저장되는 정보: 펫 ID, 이름, 외형, 성격, 특성, 생성 당시의 날씨, 기온, 시간대, 위치, 생성 일시, 펫 이미지의 로컬 파일 저장 경로
  - [x] "다음에 만나기" 버튼으로 펫을 저장하지 않고 메인 화면으로 돌아갈 수 있다
  - [x] 펫 저장 후 자동으로 메인 화면으로 전환된다

### FR-05: [펫 도감 열람 및 상세 정보 확인]

- **설명**: 사용자는 펫 도감 화면에서 수집한 모든 펫의 목록을 확인하고, 개별 펫을 선택하여 상세 정보를 열람할 수 있다.
- **수용 조건**:
  - [x] 도감 UI 진입 시 저장된 펫 목록이 카드 형태의 리스트 레이아웃으로 로드된다
  - [x] 각 펫 카드에는 이름, 수집 날짜, 주요 특성 2개, 생성 당시 날씨 아이콘과 기온이 표시된다
  - [x] 도감 헤더에 총 수집 펫 수가 표시된다
  - [x] 개별 펫 카드 선택 시 모달 형태로 상세 페이지가 표시된다
  - [x] 상세 페이지에는 펫의 모든 정보와 생성 환경 정보가 포함된다
  - [x] 수집된 펫이 없을 때 빈 상태 UI가 표시된다

### FR-06: [날씨별 배경 애니메이션]

- **설명**: 시스템은 현재 날씨 상태에 따라 메인 화면 배경에 동적 애니메이션을 표시한다.
- **수용 조건**:
  - [x] 비 오는 날씨일 때 빗방울 애니메이션(20개)이 위에서 아래로 떨어진다
  - [x] 눈 오는 날씨일 때 눈송이 애니메이션(30개)이 좌우로 흔들리며 떨어진다
  - [x] 날씨 조건(맑음, 비, 눈, 흐림)에 따라 배경 그라디언트 색상이 변경된다
  - [x] 날씨 조건 변경 시 1초 동안 부드럽게 트랜지션된다

### FR-07: [지도 기반 위치 표시 (예정)]

- **설명**: 사용자는 Google Maps API를 통해 현재 위치와 주변 펫 출현 구역을 지도에서 확인할 수 있다.
- **수용 조건**:
  - [ ] Google Maps API를 활용하여 현재 사용자 위치를 지도에 표시한다
  - [ ] 펫이 출현 가능한 반경 영역을 지도상에 마커 또는 원형으로 표시한다
  - [ ] 사용자가 해당 좌표 반경 이내로 진입하면 펫 발견 알림을 제공한다

### FR-08: [API Key 안전 관리 및 설정 기능]

- **설명**: 사용자는 설정 다이얼로그를 통해 외부 기상/AI 서비스 연동을 위한 API 키를 동적으로 구성하고 관리할 수 있다.
- **수용 조건**:
  - [x] OpenWeatherMap, Gemini, Replicate, HuggingFace API 키를 설정하는 다이얼로그를 제공한다
  - [x] API 키는 코드에 하드코딩되지 않고 SharedPreferences를 통해 기기 로컬 저장소에 안전하게 저장된다
  - [x] 텍스트 필드 우측의 표시/숨김 버튼을 통해 API 키의 공개 여부를 제어할 수 있다

### FR-09: [펫 이미지 수집 및 자동 폴백 시스템]

- **설명**: 시스템은 펫의 상세 묘사를 기반으로 이미지 프롬프트를 생성하고, AI를 활용해 고품질 펫 이미지를 무중단으로 생성한다.
- **수용 조건**:
  - [x] Gemini API를 통해 Flux 이미지 생성을 위한 상세 영문 3D 그래픽 프롬프트를 자동 생성한다
  - [x] Replicate API(Flux-Schnell)를 사용하여 펫 이미지를 비동기로 생성하고 상태를 실시간으로 폴링한다
  - [x] Replicate API 오류 혹은 토큰 부족 시 HuggingFace 무료 API(FLUX.1-schnell)로 자동 폴백하여 이미지를 생성한다
  - [x] 생성 완료된 이미지는 로컬 앱 내부 저장소(filesDir)에 영구 저장 및 캐시 관리된다

### FR-10: [도감 검색/필터링 및 편집(방생) 기능]

- **설명**: 사용자는 수집된 펫 목록을 효과적으로 필터링하고 검색할 수 있으며, 여러 마리의 펫을 자연으로 방생(삭제)할 수 있다.
- **수용 조건**:
  - [x] 이름 또는 상세 설명 텍스트를 기준으로 실시간 검색을 수행한다
  - [x] 수집 당시의 날씨(맑음, 비, 눈, 흐림) 및 시간대(아침, 오후, 저녁, 밤) 다중 필터 조건을 적용한다
  - [x] 편집 모드를 통해 다중 선택 및 전체 선택이 가능하며, 선택된 펫들을 한 번에 일괄 삭제(방생)한다
  - [x] 펫 카드의 썸네일 클릭 시 펫 이미지를 전체화면 팝업으로 크게 감상할 수 있다

### FR-11: [실시간 기상 센서 및 상세 환경 대시보드]

- **설명**: 사용자는 실시간 기상 상태 카드를 탭하여 기기 기상 센서 및 날씨 API에서 연동된 상세 환경 지표를 정밀 대시보드로 조회할 수 있다.
- **수용 조건**:
  - [x] 기기 GPS 좌표(위경도), 현재 기온, 체감 온도, 최저/최고 기온, 대기 습도, 평균 풍속, 대기 압력, 관측 시간대를 포함하는 2x4 정밀 그리드 UI를 렌더링한다
  - [x] 현재 기온, 습도, 풍속 등 핵심 지표에 대해 시각적인 가로형 프로그레스 바를 적용한다
  - [x] 평균 풍속의 세기에 비례하여 회전 속도가 달라지는 바람개비 아이콘 애니메이션을 적용한다

## 3. 비기능 요구사항 (Non-Functional Requirements)

> *어떻게 동작하는가*를 정의한다. 직접 보이지 않지만 품질을 결정한다.

### 3.1 성능 (Performance)
- 메인 화면 진입 후 1초 이내 날씨 정보 및 UI가 표시되어야 한다
- 펫 생성 API 호출은 5초 이내에 응답을 받아야 한다
- 도감 화면에서 100개 이상의 펫 목록도 원활하게 스크롤 가능해야 한다
- 애니메이션은 60fps를 유지하여 부드럽게 동작해야 한다

### 3.2 보안 (Security)
- API 키(Google Gemini, OpenWeatherMap, Replicate, HuggingFace)는 환경 변수 및 설정화면을 통해 SharedPreferences로 안전하게 관리하며 코드에 하드코딩하지 않는다
- 사용자의 위치 정보는 펫 생성 시에만 사용하며 외부 서버에 저장하지 않는다
- 로컬 DB에 저장되는 데이터는 앱 내부 저장소에만 보관한다

### 3.3 사용성 (Usability)
- 직관적인 아이콘과 한글 텍스트로 모든 기능을 설명한다
- 주요 액션 버튼(펫 생성, 도감 보기, 수집하기)은 화면 하단에 배치하여 엄지손가락으로 쉽게 접근 가능하다
- 색각 이상자를 고려하여 날씨 상태를 아이콘과 텍스트로 함께 표시한다
- 로딩 상태는 명확한 애니메이션과 텍스트로 사용자에게 피드백을 제공한다

### 3.4 호환성 (Compatibility)
- Android API 24 (Android 7.0) 이상 지원
- 세로 모드(Portrait) 기본 지원
- 다양한 화면 크기(5.5인치 ~ 6.7인치)에서 반응형으로 동작

### 3.5 기타
- 컴포넌트 기반 구조로 각 화면(Main, Generation, Collection)을 분리하여 개발
- 네트워크 연결이 끊긴 상태라도 로컬 DB에 저장된 '나만의 펫 도감' 리스트는 정상적으로 열람할 수 있도록 오프라인 조회를 보장한다.
- 추가 날씨 조건(안개, 황사 등) 쉽게 추가 가능하도록 한다.
- 펫 특성 필터링 및 정렬 기능 추가한다.
